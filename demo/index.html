Overview

What Is Slides Skill?

A Agent Skill that transforms your ideas into pixel-perfect HTML presentations. No PowerPoint. No Google Slides. Just describe your topic, pick a template, and get a self-contained HTML deck.

Works in two modes: Workflow A offers 32 hand-crafted templates for polished decks. Workflow B delivers instant self-contained output using built-in CSS themes — zero template selection needed.

Decision Guide

Which Workflow?

Use Workflow A when...

You want a visually distinctive deck. You have time to compare 3 templates. You need animated transitions, custom fonts, and a specific mood.

→ Go with Workflow A for pitches, keynotes, client presentations

Use Workflow B when...

You need a quick deck. You know which theme you want. You want a single self-contained HTML file with zero external dependencies.

→ Go with Workflow B for stand-ups, tech talks, internal reports
Workflow A Process

How It Works

1

Describe Your Topic

Tell the skill what you want to present — product launch, tech talk, research findings, team review.

2

Pick Your Template

The skill shows 3 candidates with live title-slide previews. Choose the one that fits your vibe.

3

Review & Refine

The skill builds the full deck. Add, remove, or reorder slides to match your content outline.

4

Open & Present

Get a self-contained index.html. Open in any browser — keyboard, click, or swipe to navigate.

Capabilities

Built-in Features

  • Keyboard navigation (arrow keys)
  • Click & touch / swipe support
  • Animated slide transitions
  • Staggered content reveals
  • Progress dots & slide counter
  • Fully responsive scaling
  • Self-contained single HTML file
  • No external JS dependencies
  • Custom fonts per template
  • Custom color palettes
  • Bar charts & stat layouts
  • Split layouts & quote slides
Output

Where Are Files Saved?

DEFAULT OUTPUT
~/slides/<deck-name>/index.html
PREVIEW FILES (Workflow A)
~/slides/<deck-name>/previews/
CUSTOM PATH

You can specify any output directory: "save it to ~/my-decks" or "output to /tmp/slides"
